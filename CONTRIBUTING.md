# Contributing

Thanks for your interest! This is a personal open-source project maintained by a single developer.
Contributions are welcome but please read this first.

## How to contribute

1. **Open an issue** before starting significant work — check that it aligns with the project's direction.
2. **Fork** the repo and create a branch from `main`.
3. **Write or update tests** for any behavioural change.
4. **Run the full test suite** locally and make sure everything passes.
5. **Open a pull request** against `main` using the PR template.

## Code style

Each repo documents its toolchain in `README.md` or `CLAUDE.md`. In general:

- Format before committing (`ruff format`, `prettier`, `rustfmt` — whichever applies).
- One logical change per commit, with a clear message.
- Keep PRs small and focused.
- No secrets or credentials in commits.

## Reporting bugs

Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.yml).
Include exact steps to reproduce, expected vs actual behaviour, version, and relevant logs.

## Security issues

**Do not open a public issue for vulnerabilities.** See [SECURITY.md](SECURITY.md).
